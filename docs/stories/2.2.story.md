# Story 2.2 — Sistema de Alertas e Notificações

**Epic:** 2 - Qualidade e Testes
**Status:** Done
**Priority:** P2
**Agent:** @dev

---

## Contexto

O otimizador roda em silêncio (logs em arquivo JSON). Quando detecta problemas críticos (CPA alto, budget esgotando, CTR em queda), o operador só descobre ao abrir o CLI manualmente. Para uma operação com 3 mercados e múltiplas campanhas, alertas proativos são essenciais.

## Acceptance Criteria

- [x] Sistema de alertas configurável via `config.py`
- [x] Alerta quando CPA de qualquer ad set ultrapassa threshold
- [x] Alerta quando budget diário é gasto antes das 18h (pacing ruim)
- [x] Alerta quando CTR cai mais de 30% comparado ao período anterior
- [x] Alertas enviados via pelo menos 1 canal (WhatsApp via API ou email)
- [x] Sumário diário com KPIs principais
- [x] Alertas não repetem para o mesmo ad set dentro de 24h (dedup)

## Tarefas

- [x] Criar módulo `alerts.py` com lógica de alertas
- [x] Definir thresholds configuráveis em `config.py`
- [x] Implementar canal de notificação (console + webhook genérico)
- [x] Integrar alertas no `scheduler.py` (rodar junto com otimização)
- [x] Implementar deduplicação (guardar alertas enviados em arquivo)
- [x] Criar template de sumário diário
