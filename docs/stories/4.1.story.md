# Story 4.1 — Phase 0 Foundation: Flask + SQLite + Interactive Dashboard

**Epic:** 4 - A7 Intelligence v2 Platform
**Status:** Done
**Priority:** P0
**Agent:** @dev

---

## Contexto

Evolução do projeto de CLI+HTML estático para uma aplicação web Flask com persistência de dados em SQLite. Esta é a fundação para todos os módulos futuros (Creative Intelligence, AI Coach, Budget Intelligence, Growth Score).

## Acceptance Criteria

- [x] Flask web application com factory pattern
- [x] SQLite database com schema para snapshots diários
- [x] API REST endpoints para dashboard, campanhas e histórico
- [x] Dashboard com Chart.js (gráficos interativos spend+conversions)
- [x] KPI cards com variação % vs período anterior
- [x] Botões de ação (pause/activate) inline nas campanhas
- [x] Botão refresh para atualizar dados em tempo real
- [x] Indicadores de plataforma conectada (Meta/Google)
- [x] Snapshot service para persistência de métricas históricas
- [x] Testes existentes (33) continuam passando
- [x] Módulos existentes preservados (meta_client, optimizer, alerts, etc.)

## Arquitetura Criada

```
app/
├── __init__.py .............. Flask factory (create_app)
├── db/
│   ├── schema.sql ........... 4 tabelas (daily_snapshots, campaign_snapshots, optimization_log, alerts_log)
│   └── init_db.py ........... Conexão SQLite + inicialização
├── routes/
│   ├── dashboard.py ......... GET / (Jinja template)
│   ├── api.py ............... REST API (12 endpoints)
│   └── health.py ............ GET /health
├── services/
│   ├── dashboard_service.py . Orquestra live API + DB fallback
│   ├── metrics_service.py ... Wrapper do MetaAdsClient para Flask
│   └── snapshot_service.py .. CRUD de snapshots no SQLite
├── templates/
│   ├── base.html ............ Layout base com header/footer
│   └── dashboard.html ....... Dashboard com Chart.js
└── static/
    ├── css/dashboard.css .... Estilos v2
    └── js/dashboard.js ...... Lógica interativa (charts, API calls, actions)
```

## API Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/` | Dashboard page |
| GET | `/health` | Health check |
| GET | `/api/dashboard/<range>` | Dashboard data (today/7d/30d) |
| POST | `/api/dashboard/refresh` | Refresh + store snapshot |
| GET | `/api/campaigns` | List campaigns |
| GET | `/api/campaigns/<id>/adsets` | Ad sets with insights |
| POST | `/api/campaigns/<id>/status` | Toggle status |
| POST | `/api/adsets/<id>/status` | Toggle ad set status |
| GET | `/api/history/daily` | Daily snapshots from DB |
| GET | `/api/history/campaign/<id>` | Campaign history |
| GET | `/api/comparison` | Period-over-period |
| GET | `/api/token/status` | Token validity |
| GET | `/api/platforms` | Platform connection status |

## Database Schema

- `daily_snapshots` — Account-level metrics por dia/plataforma
- `campaign_snapshots` — Campaign-level metrics por dia
- `optimization_log` — Histórico de ações do optimizer
- `alerts_log` — Histórico de alertas enviados

## Comandos

```bash
source .venv/bin/activate
python run.py                    # Dashboard em http://localhost:5050
python run.py --snapshot         # Salvar snapshot e sair
python run.py --init-db          # Inicializar banco apenas
python -m pytest tests/ -v       # Testes (33 passando)
```
