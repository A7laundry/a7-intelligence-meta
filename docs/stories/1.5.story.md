## Contexto

O `DashboardFetcher` tem dois bugs que resultam em dados incorretos no dashboard:

### Bug 1: ROAS do Meta é sempre 0
Em `dashboard_fetcher.py:83-91`, a variável `conversions_value` é inicializada como `0` mas nunca é populada a partir dos dados da API. O loop `for action in account.get("cost_per_action_type", [])` tem apenas um `pass`. Resultado: ROAS Meta é sempre 0.

### Bug 2: Daily trend sem dados Meta
Em `dashboard_fetcher.py:169`, há um comentário dizendo que Meta "doesn't have a simple daily endpoint". Na verdade, a Meta API suporta `time_increment=1` nos insights para obter dados diários. O trend chart mostra apenas dados do Google.

```python
# Meta API - daily breakdown
def get_daily_insights(self, date_preset="last_7d"):
    params = {
        "fields": "spend,impressions,clicks,actions",
        "date_preset": date_preset,
        "time_increment": 1,  # breakdown diário
    }
    result = self._request("GET", f"{self.ad_account_id}/insights", params=params)
    return result.get("data", [])
```
