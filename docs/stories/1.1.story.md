# Story 1.1 — Mover Access Token para Header Authorization

**Epic:** 1 - Segurança e Estabilidade
**Status:** Done
**Priority:** P0 - Imediato
**Agent:** @dev

---

## Contexto

O `MetaAdsClient` atualmente envia o `access_token` como query parameter na URL de todas as requisições. Isso é um risco de segurança pois:
- Tokens ficam visíveis em logs de servidor e proxy
- Podem ser capturados via referrer headers
- Ficam armazenados no histórico do navegador/ferramentas de debug

A Meta Marketing API aceita o token via header `Authorization: Bearer`.

## Acceptance Criteria

- [ ] `access_token` é enviado via header `Authorization: Bearer {token}` em todas as requisições
- [ ] Token **não** aparece mais como query parameter em nenhuma chamada
- [ ] Requisições de upload (image/video) também usam header
- [ ] `create_orlando_ads.py` também usa header (atualmente monta payload manualmente)
- [ ] Todas as chamadas de API continuam funcionando (testar com dry-run)

## Tarefas

- [x] Alterar `MetaAdsClient._request()` para usar header Authorization
- [x] Remover `params["access_token"]` do `_request()`
- [x] Alterar `upload_image()` para usar header

- [x] Refatorar `create_orlando_ads.py:create_ad()` para usar header

## Notas Técnicas

```python
# Antes (inseguro)
params["access_token"] = self.access_token

# Depois (seguro)
headers = {"Authorization": f"Bearer {self.access_token}"}
response = requests.get(url, params=params, headers=headers)
```
