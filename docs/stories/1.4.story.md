## Contexto

O access token da Meta API tem validade de ~60 dias (long-lived token). Quando expira, todo o sistema para de funcionar silenciosamente (erros 190 "Invalid OAuth access token"). Não há mecanismo de:
- Verificar validade do token
- Renovar automaticamente
- Alertar o operador antes de expirar

## Acceptance Criteria

- [ ] Na inicialização do `MetaAdsClient`, verificar se o token é válido via `GET /debug_token`
- [ ] Exibir warning se o token expira em menos de 7 dias
- [ ] Exibir erro claro se o token já expirou (em vez de erro genérico da API)
- [ ] Comando no CLI para renovar token (`python main.py --refresh-token`)
- [ ] Guardar data de expiração em `.env` ou arquivo local

## Tarefas

## Notas Técnicas

```python
def check_token_validity(self) -> dict:
    """Verifica validade do access token."""
    result = self._request("GET", "debug_token", params={
        "input_token": self.access_token,
    })
    data = result.get("data", {})
    expires_at = data.get("expires_at", 0)
    is_valid = data.get("is_valid", False)

    if not is_valid:
        print("Token expirado! Renove com: python main.py --refresh-token")
    elif expires_at > 0:
        from datetime import datetime
        expiry = datetime.fromtimestamp(expires_at)
        days_left = (expiry - datetime.now()).days
        if days_left < 7:
            print(f"Token expira em {days_left} dias ({expiry.strftime('%d/%m/%Y')})")

    return {"is_valid": is_valid, "expires_at": expires_at}
```

## Riscos

- `debug_token` endpoint requer app_id e app_secret (validar que estão no config)
- Refresh de long-lived token gera um novo token (precisa atualizar config)
