# Story 1.3 — Adicionar Paginação nas Listagens da API

**Epic:** 1 - Segurança e Estabilidade
**Status:** Done
**Priority:** P1
**Agent:** @dev

---

## Contexto

As listagens de campanhas, ad sets e ads usam `limit: 50` e ignoram o campo `paging.next` da resposta da Meta API. Contas com mais de 50 itens perdem dados silenciosamente. Isso pode causar:
- Relatórios incompletos
- Otimizador ignorando ad sets existentes
- Dashboard mostrando dados parciais

## Acceptance Criteria

- [ ] `list_campaigns()` retorna TODAS as campanhas (paginação automática)
- [ ] `list_ad_sets()` retorna TODOS os ad sets
- [ ] `list_ads()` retorna TODOS os ads
- [ ] Limite de segurança: máximo 500 itens por listagem (evitar loop infinito)
- [ ] Paginação funciona transparentemente — callers não precisam mudar

## Notas Técnicas

```python
def _paginated_request(self, endpoint: str, params: dict, max_items: int = 500) -> list:
    all_data = []
    while endpoint and len(all_data) < max_items:
        result = self._request("GET", endpoint, params=params)
        all_data.extend(result.get("data", []))
        # Meta API retorna cursor de próxima página
        paging = result.get("paging", {})
        next_url = paging.get("next")
        if next_url:
            # Próxima página usa URL completa, não endpoint relativo
            endpoint = next_url
            params = {}  # params já estão na URL
        else:
            break
    return all_data[:max_items]
```
